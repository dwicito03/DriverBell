Tentang DriverBell

DriverBell adalah aplikasi untuk memutar bell sekolah secara otomatis di komputer Windows. Setelah jadwal dibuat, DriverBell akan membunyikan audio bell sesuai jam dan hari yang sudah ditentukan.

Aplikasi ini dapat digunakan tanpa internet. DriverBell juga dibuat agar nyaman dipakai dengan keyboard dan pembaca layar seperti NVDA.

Catatan: Jadwal bell, pengaturan, dan catatan pemutaran disimpan untuk masing-masing pengguna Windows.

Kebutuhan Komputer

  • Windows 10 atau Windows 11.
  • Komputer 64-bit disarankan.
  • Tidak perlu memasang Python.
  • Tidak perlu memasang pemutar media tambahan.
  • Speaker atau perangkat audio harus aktif agar bell terdengar.

Cara Instal

  1. Buka file DriverBellSetup.exe.
  2. Ikuti petunjuk yang tampil di layar.
  3. Jika diminta memilih lokasi instalasi, biarkan lokasi bawaan jika tidak ada kebutuhan khusus.
  4. Jika ingin shortcut di desktop, centang pilihan desktop shortcut.
  5. Setelah selesai, DriverBell dapat langsung dijalankan dari Start Menu, desktop, atau pilihan Launch di akhir instalasi.

Jika muncul peringatan Windows: Pilih More info, lalu Run anyway, hanya jika installer yang digunakan berasal dari sumber resmi DriverBell.

Cara Menggunakan

Membuat Jadwal Bell

  1. Pilih tombol Tambahkan Bell.
  2. Isi nama jadwal, misalnya "Masuk Kelas", "Istirahat", atau "Pulang".
  3. Tentukan jam mulai.
  4. Pilih hari berlakunya bell.
  5. Pilih file audio yang akan diputar.
  6. Gunakan tombol Pratinjau audio bell jika ingin mencoba suaranya terlebih dahulu.
  7. Pilih Simpan.

Mengubah atau Menghapus Jadwal

  • Pilih jadwal dari daftar, lalu pilih Edit untuk mengubahnya.
  • Pilih Aktifkan atau Nonaktifkan untuk mengatur apakah jadwal tersebut berjalan.
  • Pilih Hapus untuk menghapus jadwal.
  • Jika fokus sedang berada di daftar jadwal, tekan Shift+Delete untuk menghapus jadwal yang dipilih.

Melihat Riwayat

Pilih Lihat Log untuk melihat catatan pemutaran bell. Catatan ini berguna untuk memeriksa apakah bell berhasil diputar atau tidak.

Audio dan Speaker

DriverBell dapat memutar beberapa jenis file audio, seperti mp3, wav, ogg, aac, flac, dan m4a.

Jika komputer memiliki lebih dari satu perangkat suara, misalnya speaker internal dan speaker USB, pilih perangkat yang sesuai dari menu Pengaturan.

Pengaturan

  • Zona Waktu: Pilih zona waktu yang digunakan sekolah.
  • Perangkat Keluaran: Pilih speaker atau perangkat audio yang ingin dipakai.
  • Bahasa: Pilih Bahasa Indonesia atau English.
  • Jalankan bell saat jendela ditutup: Jika aktif, DriverBell tetap berjalan di tray meskipun jendela utama ditutup.
  • Jalankan otomatis saat Windows menyala: Jika aktif, DriverBell akan dibuka otomatis saat pengguna masuk ke Windows.
  • Tampilkan dialog sambutan: Mengatur apakah pesan sambutan muncul saat aplikasi dibuka.

Autostart DriverBell tidak membuka jendela Command Prompt.

Jika Ada Masalah

Bell tidak terdengar

  • Pastikan speaker menyala dan volume tidak mute.
  • Pastikan file audio masih ada di lokasi yang dipilih.
  • Pastikan jadwal bell dalam keadaan aktif.
  • Coba gunakan tombol pratinjau untuk mengetes file audio.
  • Jika menggunakan speaker khusus, periksa pilihan Perangkat Keluaran di Pengaturan.

DriverBell belum berjalan otomatis saat Windows menyala

Buka Pengaturan, aktifkan Jalankan otomatis saat Windows menyala, lalu pilih Simpan. Jika sebelumnya memakai versi lama, matikan dulu pilihan ini, simpan, lalu aktifkan kembali dan simpan lagi.

Pembaca layar tidak membacakan daftar dengan lengkap

Pindahkan fokus keluar dari daftar, lalu masuk kembali ke daftar jadwal. Pada versi ini, DriverBell sudah menambahkan pembacaan yang lebih lengkap untuk nama jadwal, jam, hari, status, dan terakhir diputar.

Pengaturan tidak tersimpan

DriverBell menyimpan data di folder %AppData%\DriverBell. Pastikan Windows tidak memblokir akses ke folder tersebut.

About DriverBell

DriverBell is a Windows application for playing school bells automatically. After the schedule is set, DriverBell plays the selected bell audio at the chosen time and day.

The app can be used without an internet connection. It is also comfortable to use with a keyboard and screen readers such as NVDA.

Note: Bell schedules, settings, and playback logs are saved for each Windows user.

Computer Requirements

  • Windows 10 or Windows 11.
  • A 64-bit computer is recommended.
  • Python does not need to be installed.
  • No extra media player is required.
  • A speaker or audio device must be active so the bell can be heard.

Installation

  1. Open DriverBellSetup.exe.
  2. Follow the instructions on the screen.
  3. If asked to choose an install location, keep the default location unless you have a special need.
  4. If you want a desktop shortcut, enable the desktop shortcut option.
  5. After installation, DriverBell can be opened from the Start Menu, desktop, or the Launch option at the end of setup.

If Windows shows a warning: Choose More info, then Run anyway, only if the installer comes from the official DriverBell source.

How to Use

Create a Bell Schedule

  1. Choose Add Bell.
  2. Enter a schedule name, such as "Class Starts", "Break", or "Go Home".
  3. Set the start time.
  4. Select the days when the bell should run.
  5. Select the audio file to play.
  6. Use Preview the bell if you want to test the sound first.
  7. Choose Save.

Edit or Delete a Schedule

  • Select a schedule from the list, then choose Edit to change it.
  • Choose Activate or Deactivate to control whether the schedule runs.
  • Choose Delete to remove a schedule.
  • When focus is on the schedule list, press Shift+Delete to delete the selected schedule.

View Log

Choose View Log to see bell playback records. This can help you check whether a bell played successfully.

Audio and Speaker

DriverBell can play several audio file types, such as mp3, wav, ogg, aac, flac, and m4a.

If the computer has more than one sound device, such as built-in speakers and USB speakers, choose the correct one from Settings.

Settings

  • Timezone: Choose the timezone used by the school.
  • Output Device: Choose the speaker or audio device to use.
  • Language: Choose Indonesian or English.
  • Keep bells running when the window is closed: If enabled, DriverBell keeps running in the tray after the main window is closed.
  • Auto-start when Windows starts: If enabled, DriverBell opens automatically when the user signs in to Windows.
  • Show welcome dialog: Choose whether the welcome message appears when the app opens.

DriverBell autostart does not open a Command Prompt window.

Troubleshooting

The bell cannot be heard

  • Make sure the speaker is on and the volume is not muted.
  • Make sure the audio file still exists at the selected location.
  • Make sure the bell schedule is active.
  • Try the preview button to test the audio file.
  • If using a special speaker, check the Output Device setting.

DriverBell does not start automatically with Windows

Open Settings, enable Auto-start when Windows starts, then choose Save. If you used an older version before, turn this option off, save, turn it on again, and save once more.

The screen reader does not read the list completely

Move focus out of the list, then return to the schedule list. This version adds fuller reading for the schedule name, time, days, status, and last played time.

Settings are not saved

DriverBell saves data in %AppData%\DriverBell. Make sure Windows is not blocking access to that folder.
